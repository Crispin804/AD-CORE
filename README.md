## Project Overview
AD-CORE is a lightweight, responsive, single-page web dashboard application designed to analyze, track, and visualize annual business performance metrics. The platform serves as a modern digital workspace featuring interactive quantitative forms, real-time KPI generation, and clean, automated performance tracking layouts designed specifically for presentation environments.

---

## 🏗️ Project Structure
This repository contains a lightweight, decoupled static architecture. Below is a breakdown of what you will find in each primary file:

* **`index.html`** - The main application shell structure containing layout containers, tabbed content sections, entry inputs, and the custom performance graph markup.
* **`style.css`** - Custom presentation styling layers managing global theme transitions, element focus tracking, and scannable workspace fade-in animations.
* **`script.js`** - Core logical workspace engine handling real-time math calculations, data boundary validation, dynamic DOM height mutations, and global theme toggling.

## Getting Started

### Prerequisites
No advanced development build tools, runtime environments, or local web server packages are required to run this project. It runs natively as a portable static application, though an active internet connection is required to load utility styles via CDN.

### Installation & Local Setup
1. **Clone this repository** to your local machine:
```bash
   git clone [https://github.com/Crispin804/AD-CORE.git](https://github.com/Crispin804/AD-CORE.git)

```

2. Navigate directly into the project workspace directory:

```bash
   cd AD-CORE

```

3. Open `index.html` in any modern web browser to preview the live performance analytics interface instantly.

## Tech Stack

* **HTML5:** Standard semantic document skeleton defining application viewports, responsive form controls, and graph structures.
* **Tailwind CSS (via CDN):** Rapid utility-first component styling managing color scales, flexible grid alignments, and global dark-mode logic state variations.
* **Vanilla JavaScript (ES6+):** Pure DOM orchestration, runtime validation, mathematical aggregation operations, and inline style rendering.
* **Markdown:** Structural notation syntax utilized exclusively to compile standard project tracking documentation.

##  How to Collaborate

We welcome community contributions! To maintain absolute code quality, baseline structural stability, and project integrity, please adhere closely to our deployment pipeline.

### The Workflow

1. **Fork the Repository:** Create an independent, unique cloud copy of the project framework to work on.
2. **Create a Feature Branch:** Isolate your code updates cleanly away from production main branches:

```bash
   git checkout -b feature/your-feature-name

```

3. **Commit Your Changes:** Apply granular updates using precise, descriptive commit statements conforming to standard convention practices:

```bash
   git commit -m "feat: add performance indicator card variations to dashboard"

```

4. **Push to Your Branch:** Securely stage your modified branch payload updates back upstream:

```bash
   git push origin feature/your-feature-name

```

5. **Open a Pull Request (PR):** Submit your code changes clearly, specifying structural modifications, visual impacts, and linking any related tracking issue numbers.

### Contribution Guidelines

* **Maintain Strict Semantics:** Keep the underlying architecture clean and structural. Utilize native semantic elements over arbitrary nested layout containers.
* **Minimize Comments:** To keep code highly legible and professional, preserve clean self-documenting code naming practices and minimize inline comments inside script engines.
* **Documentation Maintenance:** If your branch introduces an entirely new modal container, analytical calculation, or significant design utility variation, update the **Project Structure** segment within this documentation layer immediately.

### Reporting System Issues

If you encounter code anomalies, presentation bugs, or calculation errors:

1. Review the active repository **Issues** tab history check to confirm if it has already been flagged for tracking.
2. If it is a unique bug, open a new ticket specifying an actionable title label alongside structural steps to reproduce the tracking error.
